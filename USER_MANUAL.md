## 5. Troubleshooting (User-focused)

Follow these steps if you encounter problems. Most issues can be resolved quickly.

**Problem: App won't open or crashes on launch**

1. Ensure your device meets minimum requirements and has a recent OS update.
2. Force close the app and re-open it.
3. Reboot your device.
4. If issue persists, reinstall the app from the store.

**Problem: I can't sign in**

1. Check your internet connection.
2. Make sure you're using the correct email and password.
3. Try resetting your password via **Forgot password**.
4. If you use SSO, ensure your organization's SSO service is working.

**Problem: AI responses are slow or not appearing**

1. Check your network connection and try again.
2. Wait a few seconds; occasional delays happen when the AI is processing.
3. If the problem continues, contact support with the following details:
   - Time of request
   - What you asked the assistant

**Problem: Video playback is choppy or doesn't play**

1. Check your internet connection (streaming) or ensure videos have downloaded.
2. Close other apps consuming bandwidth.
3. If downloading is stuck, retry after clearing app cache or reconnecting to Wi‑Fi.

**Problem: I don't see admin features**

1. Confirm your account has admin privileges. If not, contact an existing admin or the support team.

## Troubleshooting & FAQ

**Q1: The app won't start — "Firebase.initializeApp" error**
- **Possible cause**: `lib/firebase_options.dart` mismatch or missing platform config (`google-services.json` or `GoogleService-Info.plist`).
- **Fix**: Ensure `lib/firebase_options.dart` matches your Firebase project or re-run `flutterfire configure`. For Android/iOS add platform files.

**Q2: Login fails with authentication error**
- **Possible cause**: Firebase project not set up correctly, or Firestore/Firebase rules blocking sign-in.
- **Fix**: Verify Firebase project credentials, check `lib/firebase_options.dart`, open Firebase console to review Authentication settings.

**Q3: AI responses fail or are blank**
- **Possible cause**: AI API key missing or rate-limited; in development the app might call AI directly from the client.
- **Fix**: Check whether the app was configured to use a server proxy for AI requests. If using client side, ensure the API key is available to the app (development only). For production, implement and point `AiService` to a secure proxy.

**Q4: Video playback doesn't work**
- **Possible cause**: Missing assets or `assets/videos/` not declared in `pubspec.yaml`.
- **Fix**: Verify `pubspec.yaml` includes `assets/videos/` and that files exist under `assets/videos/`.

**Q5: I can't run admin scripts (Node errors)**
- **Fix**: Run `npm ci` in the `tools/` folder and ensure Node version is compatible (use `nvm` or Node 16+ recommended).

General troubleshooting steps
1. Run `flutter doctor` and resolve any reported issues.
2. Run `flutter clean` then `flutter pub get`.
3. Check console logs (`flutter run`) for stack traces and messages.
